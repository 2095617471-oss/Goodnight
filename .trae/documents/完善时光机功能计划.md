# 完善时光机功能计划

## 1. 目标与范围
完善「时光机」模块功能，提升用户的情感体验。主要包括：
- **逻辑优化**：增加未来日期选择限制、信件解封倒计时、列表空状态提示。
- **视觉与动画升级**：引入信封样式的 UI 设计，增加寄信动画和信件入场动画，增强“时空投递”的仪式感。

## 2. 当前状态分析
- `index.html` 中的时光机表单和列表目前仅使用基础卡片样式，缺乏“时空信件”的代入感。
- `app.js` 中的存信逻辑允许选择过去的日期；未到期信件仅显示静态的“尚未到达”提示，没有直观的倒计时；列表为空时直接留白，无引导提示。

## 3. 拟修改内容

### 3.1 逻辑与数据处理 (`js/app.js`)
- **日期限制**：在应用初始化时，获取 `#future-date` 元素，将其 `min` 属性设置为“明天”的日期（`YYYY-MM-DD`），强制用户只能给未来的自己写信。
- **空状态处理**：在 `renderFutureMessages` 中，如果 `messages.length === 0`，则向 `#future-messages-list` 中注入空状态 HTML 结构。
- **倒计时计算**：在渲染未到期信件时，计算 `targetDate` 与当前日期的毫秒差，转换为天数，展示“还有 X 天解封”。
- **动画衔接**：修改存入时光机的点击事件，在保存数据后，先为表单添加 `.sending` 动画类，使用 `setTimeout` 延迟 500ms 等待动画完成后，再清空输入框并重新渲染列表。

### 3.2 结构与类名调整 (`index.html`)
- 在时光机表单区（`.future-form`）增加信封样式的结构，例如邮票图标的装饰元素。
- 将保存按钮的文案调整为“投递信件”，并加上发送图标。

### 3.3 视觉与动画样式 (`css/style.css`)
- **表单视觉 (`.future-form`)**：
  - 增加信封的特征，例如四周虚线边框、类似信纸的轻微背景色区分。
  - 相对定位以便放置装饰性元素。
- **信件卡片 (`.future-message-card`)**：
  - 添加未解封 (`.sealed`) 和已解封 (`.unsealed`) 的视觉区分。
  - 未解封状态：虚化文本，增加锁图标，色调偏暗。
  - 已解封状态：明亮，突出信件内容。
- **动画效果 (`@keyframes`)**：
  - `@keyframes sendLetter`：寄信时，表单内容轻微缩小并向上淡出的动画（或纸飞机飞出动画）。
  - `@keyframes slideInUp`：信件列表中新卡片渲染时的淡入上浮动画。

## 4. 假设与决策
- **动画时间**：寄信动画设定为 0.5s，不会过度阻塞用户的操作流。
- **倒计时计算**：采用 `Math.ceil((target - now) / 86400000)` 的方式计算天数，确保即便是明天解封，也会显示“还有 1 天解封”。
- **依赖**：不引入新的库，利用现有的 CSS 变量体系和 RemixIcon。

## 5. 验证步骤
1. 刷新页面，进入时光机模块。
2. 点击日期选择器，验证是否只能选择明天的日期及以后。
3. 观察无信件时，是否正确显示空状态引导。
4. 填写信件并点击投递，观察表单寄出动画和列表更新动画。
5. 检查未到期信件的倒计时计算是否准确（例如选明天，应显示还有 1 天）。
6. 检查已到期/过去测试数据信件（如有）是否正确展示内容。
