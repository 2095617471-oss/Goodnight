# 新增纯前端登录功能开发计划

## 一、需求摘要
为现有的“深夜情绪树洞日记”网站新增一个纯前端的登录与注册系统。要求完全基于浏览器的 `localStorage` 实现，不依赖后端与数据库；支持账号密码的注册、登录和退出；注册时需校验用户名冲突和密码二次确认；登录状态有效时才能访问日记等核心功能；整体 UI 保持深夜暗色治愈风（圆角、柔和文字、悬浮动效）。

## 二、当前状态分析
- **架构**：纯前端单页应用（SPA）。
- **数据存储**：封装了统一的 `Storage` 对象，数据（如草稿、日记、时光机、密码锁等）以 `goodnight_` 为前缀直接存储在 `localStorage` 中，当前为单用户模式。
- **页面结构**：存在 `#lock-screen`（密码锁层）和 `#app-container`（主应用层）。缺少登录/注册专用的阻断层。

## 三、具体实施方案

### 1. `js/app.js` 逻辑改造 (核心)
- **改造 `Storage` 数据隔离**：
  - 修改 `Storage.get`, `Storage.set`, `Storage.remove` 方法。
  - 对于全局白名单 Key（如 `users`, `currentUser`），直接存取。
  - 对于其他业务 Key（如 `diaries`, `draft`, `future_msgs`, `password`），动态加上当前登录用户的前缀（例如 `goodnight_alice_diaries`），从而实现**多账号数据完全隔离**。
- **新增 `Auth` 认证模块**：
  - 实现 `init()` 初始化 DOM 绑定。
  - 实现 `switchTab()` 切换登录与注册表单。
  - 实现 `register()`：校验输入为空、密码不一致、用户名已存在（从 `Storage.get('users')` 读取），成功后存入用户列表并自动登录。
  - 实现 `login()`：比对用户名与密码，失败提示错误，成功则设置 `currentUser` 并展示主界面。
  - 实现 `logout()`：清除 `currentUser` 并刷新页面。
  - 实现 `checkLoginStatus()`：应用启动时检查登录状态，未登录则拦截并显示 `#auth-screen`。
- **调整启动流程**：在 `init()` 最前面调用 `Auth.init()`，确保鉴权拦截优于其他模块渲染。

### 2. `index.html` 结构修改
- **新增 `#auth-screen` 登录注册层**：
  - 放置在 `body` 顶层（与 `#lock-screen` 同级）。
  - 内部包含 `.auth-container`，提供顶部 Tab（登录/注册切换）。
  - **登录表单**：用户名输入框、密码输入框、登录按钮、错误提示区。
  - **注册表单**：用户名输入框、密码输入框、确认密码输入框、注册按钮、错误提示区。
- **新增退出登录按钮**：在左侧导航栏底部（`.nav-footer`）的“按 Esc 一键隐藏”上方，添加一个带有图标的“退出登录”按钮。

### 3. `css/style.css` 样式修改
- **认证屏容器 (`#auth-screen`)**：
  - 全屏遮罩，背景使用全局暗色 `var(--bg-color)`，层级 `z-index: 9998`。
  - `.auth-container` 采用大圆角 `border-radius: 24px`，背景色 `var(--surface-color)`，并添加柔和的悬浮阴影效果。
- **表单元素样式**：
  - Tab 切换按钮样式（激活状态带有主题色下划线）。
  - 输入框继承现有的暗色极简风格，Focus 时边框变为 `var(--accent-color)`。
  - 错误提示文本（`.error-text`）使用 `var(--danger-color)` 并带有关闭间距。
- **退出登录按钮**：复用现有的 `.back-btn` 样式，增加宽度 100% 适配底栏布局，Hover 时变色。

## 四、假设与决策
- **安全性说明**：由于是纯前端项目，密码明文（或简单存储）保存在 `localStorage` 中，仅防君子不防黑客。这符合纯前端演示与“完全不使用后端”的要求。
- **数据迁移**：新的隔离机制启用后，之前未登录时留下的全局数据将被留存在缓存中，新注册的用户将拥有一个全新的空白树洞。

## 五、验收标准 (Verification)
1. **页面拦截**：打开页面时，若未登录，直接显示登录界面，无法看到日记内容。
2. **注册校验**：
   - 密码与确认密码不一致时报错。
   - 注册已存在的用户名时报错。
   - 注册成功后自动进入主系统。
3. **登录功能**：输入错误账号/密码报错，输入正确后成功进入。
4. **数据隔离**：注册账号 A 并写下一篇日记；退出后注册账号 B，日记列表应为空。
5. **退出登录**：点击左下角“退出登录”，页面立即回到登录屏。
