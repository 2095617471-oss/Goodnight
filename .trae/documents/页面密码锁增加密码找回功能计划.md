# 页面密码锁增加密码找回功能计划

## 1. 目标与需求分析
用户希望在页面密码锁屏界面，增加一个“密码找回”功能。
由于应用本身有一套基于“用户名+密码”的账号登录体系，当用户忘记了“页面密码锁”时，最安全且合理的找回方式是：要求用户输入其**账号的登录密码**进行身份验证。验证通过后，系统自动清除页面密码锁并解锁应用。

## 2. 现状分析
- **HTML 结构**：`index.html` 中的 `#lock-screen` 包含了锁屏界面的密码输入框和解锁按钮，目前缺少“忘记密码”的交互入口。
- **JS 逻辑**：`js/app.js` 中：
  - 通过 `Storage.get('password')` 和 `Storage.remove('password')` 来读取和清除页面锁密码。
  - 已经存在 `validatePassword(password)` 函数，可以直接用于验证当前登录账号的密码。
  - 存在 `initPrivacy()` 用于初始化与锁屏相关的事件。

## 3. 具体修改方案

### 3.1 修改 HTML (`index.html`)
在锁屏界面（`<div class="lock-container">`）内部的错误提示 `<p id="lock-error">` 下方，新增一个“忘记密码”的文本按钮，复用已有的样式类：
```html
<p id="forgot-lock-password" class="forgot-password-link" style="margin-top: 15px;">忘记页面密码？</p>
```

### 3.2 修改 JS DOM 引用 (`js/app.js`)
在文件顶部的 `elements` 对象定义中，加入对新元素的引用：
```javascript
forgotLockPassword: document.getElementById('forgot-lock-password'),
```

### 3.3 增加找回逻辑 (`js/app.js`)
在 `initPrivacy()` 函数内，为 `forgotLockPassword` 绑定点击事件：
1. 弹出输入框：`prompt("请输入您的账号登录密码以验证身份并清除页面锁密码：")`。
2. 若用户输入了密码，则调用 `validatePassword(accountPwd)` 验证：
   - **验证成功**：
     - 执行 `Storage.remove('password')` 彻底清除页面锁密码。
     - 隐藏锁屏界面 (`lockScreen`)，显示主应用容器 (`appContainer`)。
     - 清空原输入框内容并隐藏可能的错误提示。
     - 弹窗提示“页面密码锁已清除并解锁。”。
     - 调用 `checkFutureMessagesPopup()` 检查是否有未来的信件弹出。
   - **验证失败**：
     - 使用 `alert("账号密码错误，无法验证身份。")` 提示用户。

## 4. 验证与测试
1. 在“设置与安全”中设置一个页面密码锁。
2. 刷新页面，进入锁屏界面。
3. 点击“忘记页面密码？”按钮，随便输入一个错误的账号密码，确认弹出错误提示且未能解锁。
4. 再次点击，输入当前账号正确的登录密码，确认提示成功并直接进入应用页面。
5. 前往“设置与安全”页面，确认密码锁状态已变回“未设置”状态。
