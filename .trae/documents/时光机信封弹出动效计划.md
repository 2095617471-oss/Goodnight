# 增加时光机信封弹出动效功能计划

## 1. 目标与范围
在用户打开网站（且已登录/解锁）时，检查「时光机」中是否有解封日期为今天的信件。如果有，且是首次打开，则以淡入/飘入的动画弹出一个“信封”。用户点击可拆开阅读，阅读后关闭。内置一条“今天”解封的测试数据，方便直接验证效果。

## 2. 当前状态分析
- 时光机信件数据存在 `localStorage` 的 `future_msgs` 中。
- 目前信件到期后，只会在「时光机」列表里变更为“已送达”状态，缺乏主动提醒和“收信”的仪式感。
- `app.js` 的 `initializeAppAfterLogin` 和 `unlockApp` 控制了进入主界面的时机，是触发弹窗的理想位置。

## 3. 拟修改内容

### 3.1 增加弹窗结构 (`index.html`)
在 `<body>` 根层级（如 `#quote-toast` 前后）增加信封弹窗的 HTML 结构 `#letter-popup-modal`，包含：
- 半透明遮罩层 (`.letter-popup-overlay`)
- 信封容器 (`.letter-popup-container` 和 `.letter-envelope`)
- 封面元素（显示图标和“有一封来自过去的信”提示）
- 信纸元素（包含写信日期、信件内容和“收下这份心意”的关闭按钮）

### 3.2 增加弹窗与动画样式 (`css/style.css`)
- **弹窗基础**：固定定位、高层级 (`z-index`)、居中对齐。
- **入场动画 (`.shake-in`)**：使用 `@keyframes` 结合 `translateY`、`rotateX` 和 `opacity` 实现由下向上的淡入上浮，配合轻微的上下浮动/抖动效果，模拟真实的信件递送感。
- **拆开动画 (`.opened`)**：点击信封时，封面淡出，信纸从信封中向上滑出并放大，信封主体下沉，实现平滑的“拆信”过渡。

### 3.3 逻辑处理与测试数据 (`js/app.js`)
- **弹窗逻辑**：新增 `checkFutureMessagesPopup()` 函数。查找 `future_msgs` 中 `targetDate <= today` 且 `!has_popped_up` 的信件。若存在，取第一封进行弹出，并立即将该信件的 `has_popped_up` 设为 `true` 以防重复弹出。
- **触发时机**：在 `initializeAppAfterLogin` 中判断未被密码锁拦截时调用，以及在 `unlockApp` 密码解锁成功后调用。
- **弹窗交互**：新增 `showLetterPopup(msg)` 函数，控制弹窗展示、点击拆信的类名切换 (`.opened`)，以及点击关闭按钮隐藏弹窗的事件。
- **测试数据修改**：
  - 修改 `injectWangSiAnData`，为前天（已到期）的测试数据加上 `has_popped_up: true` 标记，避免干扰。
  - 新增第三条测试数据，`targetDate` 设为今天（`now`），确保用户一刷新页面就能立即看到弹出效果。

## 4. 假设与决策
- **动画时长**：入场淡入 1s，上下悬浮循环周期 4s，拆信过渡 0.6s，确保整体效果柔和不突兀。
- **防重复触发**：通过在原信件数据上追加 `has_popped_up: true` 属性来记录弹出状态，纯本地维护，无需新增单独的存储 key。
- **多封信件处理**：若有多封未读，每次打开网站只弹出一封（通过 `unpoppedMessages[0]` 控制），避免多个弹窗堆叠影响体验。

## 5. 验证步骤
1. 在登录页点击 5 次“欢迎回来”触发测试数据注入。
2. 页面刷新自动登录后，观察屏幕中央是否弹出一个悬浮抖动的信封。
3. 点击信封，观察是否平滑展示信纸和信件内容。
4. 点击关闭按钮，弹窗消失回到主页。
5. 再次刷新页面，确认信封不会再次弹出。
