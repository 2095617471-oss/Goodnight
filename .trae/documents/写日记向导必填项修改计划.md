# 写日记向导必填项修改计划

## 1. 目标与需求分析
用户希望在“写日记”分步向导中，将“当时和谁在一起”（第三步）与“当时你在做什么”（第四步）设置为必填项，即输入框不能为空。如果用户未填写内容直接点击“下一步”，系统需要给出提示并拦截操作。

## 2. 现状分析
当前代码结构分析：
- 文件：`js/app.js`
- 验证逻辑函数：`validateCurrentStep(stepDiv)`
- 现有逻辑：目前只对 `wizard-step-1` (时间和时段) 以及 `wizard-step-2` (地点 `wizard-location`) 进行了必填拦截，拦截提示使用 `showToast`。
- 第三步 (`wizard-step-3`) 输入框 ID 为 `wizard-people`。
- 第四步 (`wizard-step-4`) 输入框 ID 为 `wizard-action`。

## 3. 具体修改方案

### 3.1 修改 `validateCurrentStep` 验证逻辑
在 `js/app.js` 中的 `validateCurrentStep` 函数里，追加对 `wizard-step-3` 和 `wizard-step-4` 的空值校验。

- **增加步骤3的校验**：
  获取 `wizard-people` 的输入值，去除首尾空格。如果为空，则调用 `showToast('请填写人物，这是必填项')`，并返回 `false`。
- **增加步骤4的校验**：
  获取 `wizard-action` 的输入值，去除首尾空格。如果为空，则调用 `showToast('请填写事件，这是必填项')`，并返回 `false`。

## 4. 验证与测试
- 启动应用并进入“写日记”向导。
- 填写第一步时间、第二步地点后进入第三步。
- 保持第三步（当时和谁在一起）为空，点击“下一步”，验证是否拦截并弹出 Toast 提示。
- 填写内容后进入第四步。
- 保持第四步（当时你在做什么）为空，点击“下一步”，验证是否拦截并弹出 Toast 提示。
- 填写内容后，确认可以正常进入下一步进行情绪打分。
