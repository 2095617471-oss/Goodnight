# 写日记向导第一步排版修改计划

## 1. 目标与需求分析
用户反馈“你要记录的情绪发生在什么时候？”这一步（向导的第一步）排版不正常，希望将其改为“上下”排版。
从当前的截图和 CSS 代码可以看出，目前在大屏幕（`min-width: 860px`）下，日期选择和时段选择是被并排（左右）显示的（`grid-template-columns: 1fr 1fr;`）。用户希望即使在大屏幕下，也保持“日期”在上方，“时段”在下方的垂直堆叠排版。

## 2. 现状分析
- 涉及文件：`css/style.css`
- 结构：
  - `.wheel-picker-container` 默认在移动端是 `grid-template-columns: 1fr;`（上下排版）。
  - 在第 1953 行的 `@media (min-width: 860px)` 媒体查询中，有针对大屏幕的特殊设置：
    ```css
    #wizard-step-1 .wheel-picker-container {
        grid-template-columns: 1fr 1fr;
        align-items: start;
    }
    ```
  这就导致了在大屏幕下，日期和时段变成了左右并排的布局。

## 3. 具体修改方案

### 3.1 移除大屏幕下的分栏设置 (`css/style.css`)
- 找到 `css/style.css` 中第 1952 行附近的 `@media (min-width: 860px)` 块。
- 删除内部关于 `#wizard-step-1 .wheel-picker-container` 改变 `grid-template-columns` 的样式覆盖。
- 这样，`.wheel-picker-container` 将始终使用其基础定义中的 `grid-template-columns: 1fr;`，从而在任何屏幕尺寸下都保持“日期”和“时段”为上下垂直排版。

## 4. 验证与测试
- 移除对应 CSS 代码后，在桌面端大屏幕下打开“写日记”的第一步。
- 观察“日期选择”与“时段选择”是否已经变成了上下排版，不再是左右并排。
