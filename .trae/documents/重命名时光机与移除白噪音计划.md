# 移除“白噪音播放器”并重命名模块计划

## 一、需求摘要
按照用户要求，删除“治愈陪伴”模块中的“白噪音播放器”板块，只保留“给未来的自己”板块，最后将整个模块在左侧导航和页面标题上的名字从“治愈陪伴”修改为“时光机”。

## 二、当前状态分析
1. **导航与模块标题 (`index.html`)**：
   - 侧边栏有一个 `<li data-tab="healing">`，显示图标和文字“治愈陪伴”。
   - 内容区有一个 `<section id="tab-healing">`，内部包含 `<h2>治愈陪伴</h2>`，以及两个主要的卡片容器：`.noise-card`（白噪音）和 `.future-card`（给未来的自己）。
2. **样式 (`css/style.css`)**：
   - 有一系列专门为白噪音播放器编写的样式，如 `.noise-controls`, `.noise-btn`, `.volume-control` 等。
3. **逻辑 (`js/app.js`)**：
   - `elements` 对象中缓存了 `.noise-btn` 和音量控制条。
   - 监听 `Escape` 键的一键隐藏功能中包含了暂停所有 `<audio>` 的逻辑。
   - `initHealing()` 函数中包含了点击播放/暂停白噪音以及控制音量的业务逻辑。

## 三、具体实施方案

### 1. 修改 HTML (`index.html`)
- **导航栏更新**：将 `<li data-tab="healing">` 内部的图标改为 `<i class="ri-time-line"></i>`，文字修改为 `<span>时光机</span>`。
- **标题更新**：将 `<section id="tab-healing">` 的标题 `<h2>治愈陪伴</h2>` 修改为 `<h2>时光机</h2>`。
- **删除冗余代码**：彻底移除 `<div class="card noise-card">` 及其内部所有的 HTML 结构（包括音频按钮、音量条和 `<audio>` 标签）。

### 2. 清理 CSS (`css/style.css`)
- **修改注释**：将 `/* --- 治愈陪伴 & 设置模块 --- */` 更改为 `/* --- 时光机 & 设置模块 --- */`。
- **删除冗余样式**：删除从 `.noise-controls` 到 `.volume-control input` 之间的所有不再使用的 CSS 类，保持代码整洁。

### 3. 重构 JS 逻辑 (`js/app.js`)
- **清理 DOM 缓存**：在 `elements` 对象中，将注释 `// 治愈` 改为 `// 时光机`，并删除 `noiseBtns` 和 `globalVolume` 变量。
- **清理播放逻辑**：在 `initHealing()` 函数中，删除关于“白噪音控制”和“全局音量控制”的所有逻辑。
- **清理一键隐藏逻辑**：在 `keydown` 监听事件中，删除按下 `Esc` 键时调用 `audio.pause()` 和移除 `playing` 类的相关代码。

## 四、假设与决策
- **标识符保留**：为了减少潜在的连带 Bug，HTML 中 `data-tab="healing"` 和 `id="tab-healing"` 以及 JS 中 `initHealing()` 的底层命名保持不变，只修改面向用户的展示文本“时光机”。
- **单卡片布局**：由于删除了上方的白噪音卡片，“给未来的自己”将成为该模块唯一的内容卡片，由于之前有外层 `section` 的包裹和独立的 `.card` 样式，删除上方卡片后排版依然会自动适应，无需额外修改布局代码。

## 五、验收标准 (Verification)
1. 页面左侧导航栏原来的“治愈陪伴”变为“时光机”，图标为时钟/历史相关图标。
2. 点击进入该模块，页面大标题显示“时光机”。
3. 页面内不再有“白噪音播放器”及相关音频控制按钮，仅展示“给未来的自己”表单和历史留言列表。
4. 检查浏览器控制台，确认移除相关 DOM 元素后没有抛出 JS 报错（特别是一键隐藏、切页等交互）。
