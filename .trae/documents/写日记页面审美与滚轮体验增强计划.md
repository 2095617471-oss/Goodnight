# 写日记页面审美与滚轮体验增强计划

## Summary

对「写日记」页（分步向导 + 结果页 + 进入编辑器前的流程）做审美与比例优化：减少空洞留白、增强层次与对齐、让信息结构更清晰；同时保留现有“年月日滚轮式选择（select + wheel-picker 样式）”的交互方式，并新增轻量动效与即时反馈，提升用户体验感。

## Current State Analysis

### 页面结构（现状）

- 写日记入口在 `#tab-write`，先展示 `#wizard-container`（分步向导），完成后进入 `#editor-container`（编辑器）。[index.html](file:///c:/Users/lenovo/Desktop/GoodNight/index.html#L151-L347)
- 第一步（时间）由三段 `<select class="wheel-picker">` 组成日期选择器，并配合 `.time-period-option` 选择时段。[index.html](file:///c:/Users/lenovo/Desktop/GoodNight/index.html#L155-L197)
- 向导切换由 `hidden` 类控制；下一步/上一步按钮事件在 `initEditor()` 内绑定。[app.js](file:///c:/Users/lenovo/Desktop/GoodNight/js/app.js#L891-L965)
- 年月联动通过 `updateDays()` 重新生成日选项实现。[app.js](file:///c:/Users/lenovo/Desktop/GoodNight/js/app.js#L1011-L1080)

### 视觉问题（从现有实现可推导）

- `.wizard-area` 使用 `min-height: 60vh` 且整体居中（`justify-content: center; align-items: center`），在常见窗口高度下容易产生“上方标题过大 + 中间空白过多 + 内容集中在中部”的比例失衡，观感偏“空且散”。[style.css](file:///c:/Users/lenovo/Desktop/GoodNight/css/style.css#L365-L423)
- 第一步的日期与时段是纵向堆叠（`.wheel-picker-container` 为 column），信息密度低、下方“下一步”离内容较远；时段卡片边框较粗（2px）且状态反馈较弱，整体显得“板”。[style.css](file:///c:/Users/lenovo/Desktop/GoodNight/css/style.css#L1407-L1514)
- “滚轮选择器”本质是原生 `<select>`，目前只有基础边框与 focus 边框色变化，缺少更明确的交互提示（例如选中后的即时反馈、轻微动效、清晰的层次/高光）。[style.css](file:///c:/Users/lenovo/Desktop/GoodNight/css/style.css#L1407-L1462)
- 结果页存在大量内联样式（style=...），难以与整体风格统一维护，且卡片层次可进一步提升。[index.html](file:///c:/Users/lenovo/Desktop/GoodNight/index.html#L307-L322)

## Proposed Changes

### A. 写日记向导的比例与结构优化（不改变功能）

**目标**：让第 1 步在视觉上更像“一个卡片化的引导面板”，上下信息更聚合、左右对齐更规整；在桌面端提升信息密度，在移动端保持易用。

- 文件：`css/style.css`
  - 调整 `.wizard-area`：从“垂直居中”改为“顶部更自然的版式”，采用顶部内边距 + 适度最大宽度，减少空洞留白。
  - 强化 `.wizard-step`：增加卡片层次（背景、边框、阴影/渐变），并减少 `h2` 的 `margin-bottom`，让内容更紧凑。
  - 第一步布局改为“宽屏两列，窄屏一列”：
    - 宽屏：日期选择（左）与时段选择（右）并排；按钮区贴近内容。
    - 窄屏：保持现有纵向堆叠，避免拥挤。

### B. 保留滚轮选择方式但增强体验（交互提示 + 微动效）

**目标**：不自绘、不替换控件类型，仍使用现有 `<select class="wheel-picker">`；通过样式与轻量 JS 增强“在选、已选、变更”的反馈。

- 文件：`index.html`
  - 在第 1 步增加一个“即时摘要行”（例如：`已选择：2026年5月3日 · 晚上`），用于实时反馈用户当前选择结果，强化确定感。
  -（可选）在 `#wizard-container` 顶部增加“步骤进度条/圆点”（例如 1/6），降低迷失感。
- 文件：`css/style.css`
  - 为 `.wheel-picker` 增加更明显的交互 affordance：
    - 更柔和的边框与内阴影，hover 时轻微抬升，focus 时出现柔和外发光；
    - 下拉提示（例如右侧小箭头），让它更像“可拨动的滚轮控件”。
  - 为日期/时段“变更”提供短暂动效类（例如 `is-changed`）：轻微缩放/发光 150–250ms，用于强化“我刚刚选中了/改变了”。
  - 时段卡片（`.time-period-option`）增强选中态：
    - 选中时增加背景渐变与阴影、图标微上浮；
    - hover 反馈更细腻（边框变色 + 轻微位移）。
- 文件：`js/app.js`
  - 在 `initDatePicker()` 中为 `#wizard-year/#wizard-month/#wizard-day` 的 `change` 事件补充：
    - 更新“即时摘要行”内容；
    - 对发生变化的控件添加 `is-changed` 类并在短时间后移除（动效触发器）。
  - 在时段选择 click 回调里同样更新摘要与动效。
  - 若引入“步骤进度条”，在向导步切换的关键点调用 `updateWizardProgress(step)` 更新 UI（只做最小侵入，复用现有切换逻辑，不改业务数据结构）。

### C. 结果页与进入编辑器前的视觉统一（小范围）

**目标**：只处理与“写日记流程审美”直接相关的部分，减少内联样式，使整体更协调。

- 文件：`index.html` + `css/style.css`
  - 把结果页内联 style 抽成类（例如 `.emotion-result-card`、`.generated-summary-box` 等），统一字体、间距、边框与背景。
  - “开始记录”按钮与“下一步”按钮风格统一（尺寸、圆角、间距）。

## Assumptions & Decisions

- 仅做写日记流程 UI/UX 优化，不改变任何数据结构（`state.wizardData` / Storage）与业务流程（步骤顺序、校验规则、保存逻辑）。
- “滚轮式选择年月日”保持现有 `<select>` 实现，不引入外部库、不自绘触控滚轮；增强体验以 CSS 微动效 + 即时摘要反馈为主。
- 动效以短时、低干扰为原则：不影响性能、不造成眩晕，优先使用 `transform/opacity`，并兼容 `prefers-reduced-motion`（若项目已有相关处理则复用，否则做最小补充）。

## Verification Steps

1. 桌面端：进入「写日记」第 1 步，确认标题、日期、时段、按钮整体比例更协调；日期与时段在宽屏呈两列且对齐自然。
2. 第 1 步：切换年/月/日时，“即时摘要行”实时变化；控件出现短暂反馈动效；日期联动更新日选项正常（`updateDays()` 不受影响）。
3. 第 1 步：选择时段时选中态明显、hover 手感更强；摘要同步更新。
4. 小屏/窄屏：第 1 步自动变为纵向堆叠，控件不挤压、不溢出；按钮可见且易点。
5. 走完整流程（step-1→…→result→editor）：切换动画与布局稳定，无错位；结果页样式与整体一致，且“开始记录”进入编辑器正常。
