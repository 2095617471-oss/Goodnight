# 「往日回音」心情日历功能开发计划

## 一、需求摘要
在“往日回音”模块（历史日记页）新增一个**心情日历**功能。日历以网格形式展示当月每一天，并根据当天日记的情绪打分（喜、怒、哀、惧）为日期格子填充不同颜色。要求纯前端实现、响应式布局、符合深夜暗色治愈风，并支持悬停显示情绪详情和点击跳转至该日记。

## 二、当前状态分析
- **页面结构**：`index.html` 中的 `#tab-history` 模块目前仅包含一个标题、搜索框和垂直排列的日记列表（`#diary-list`）。
- **数据结构**：日记数据存储在 `localStorage` 的 `diaries` 数组中。每条日记包含 `date`（ISO时间戳字符串）、`scores`（喜怒哀惧1-3分）和 `matchedEmotion`（综合情绪结果）等字段。
- **颜色规范**：现有应用已有一套情绪颜色变量（如 `--mood-calm`, `--mood-sad`, `--mood-anxious` 等），需要在日历中复用或补充对应的高饱和度/低饱和度色值以适应日历块。

## 三、具体实施方案

### 1. `index.html` 结构修改
在 `#tab-history` 的 `.history-header` 下方、`#diary-list` 上方，新增日历组件容器：
- **日历头部 (`.calendar-header`)**：包含左/右切换月份的按钮（使用 `ri-arrow-left-s-line` 等图标）和当前月份显示（如“2026年5月 心情日历”）。
- **日历主体 (`.calendar-body`)**：
  - 星期表头（周一至周日）。
  - 日期网格容器 (`#calendar-grid`)，用于 JS 动态生成日期格子。
- **月度总结 (`.calendar-summary`)**：在日历下方显示本月最常见的心情标签。

### 2. `css/style.css` 样式修改
- **日历布局**：使用 `display: grid; grid-template-columns: repeat(7, 1fr);` 实现整齐的 7 列日历网格。
- **日期格子 (`.cal-day`)**：
  - 基础样式：暗色背景 (`rgba(255,255,255,0.03)`)、圆角、居中对齐、文字颜色柔和。
  - **情绪颜色填充**：为不同主导情绪定义类名（如 `.mood-bg-joy`, `.mood-bg-anger`, `.mood-bg-sorrow`, `.mood-bg-fear`, `.mood-bg-mixed`），背景色使用对应的全局情绪颜色，并适当降低透明度或调整亮度以保证文字可读性。
- **交互动效**：
  - Hover 时格子轻微放大（`transform: scale(1.1)`），边框高亮。
  - 使用原生 `title` 属性或自定义 Tooltip 实现悬停显示详细分数和标签。
- **响应式**：移动端缩小格子 padding 和字号，保证 7 列不溢出屏幕。

### 3. `js/app.js` 逻辑修改
- **状态管理**：在 `state` 对象中新增 `calendarDate`，默认为当前日期（用于追踪当前查看的月份）。
- **日历渲染函数 (`renderCalendar`)**：
  1. 获取 `state.calendarDate` 所在月份的第一天是星期几、总天数。
  2. 读取 `Storage.get('diaries', [])`。
  3. **数据聚合**：构建一个以 `YYYY-MM-DD` 为 Key 的字典，Value 为该日期的日记数据。若一天有多篇，取最后（或最高分）的一篇作为代表。
  4. 生成网格 DOM：填补月初空白格，循环生成每一天的格子。
  5. **颜色判定**：判断当天的 `scores` 中哪个维度分数最高（与生成卡片边框逻辑一致），赋予对应的颜色类。
  6. **事件绑定**：
     - 为有日记的格子添加 `title` 属性（如：“综合情绪：平静 | 喜:2 怒:1 哀:1 惧:1”）。
     - 为有日记的格子绑定点击事件：页面平滑滚动到 `#diary-list` 中对应的日记卡片，并给该卡片添加短暂的高亮闪烁动画。
- **月度统计**：在 `renderCalendar` 末尾，统计本月出现次数最多的 `matchedEmotion.emotion`，更新到 `.calendar-summary` 元素中。
- **月份切换**：绑定左右按钮事件，修改 `state.calendarDate` 并重新调用 `renderCalendar`。
- **初始化集成**：在 `initHistory()` 中调用 `renderCalendar()`，并在保存新日记后也触发更新。

## 四、假设与决策
- **一天多篇日记处理**：为保持日历界面的整洁，如果用户在同一天写了多篇日记，日历格子的颜色和 Hover 提示将以**当天最后一篇**（最新）日记的情绪为准。
- **跳转交互**：由于目前没有独立的“日记详情页”（所有日记都在列表中展开展示），点击日历格子的行为将设计为**锚点跳转（Scroll Into View）**，即页面自动向下滚动到列表中对应的那篇日记，并闪烁提示。

## 五、验收标准 (Verification)
1. **视觉展示**：往日回音页面顶部正确显示网格日历，暗色治愈风，有日记的日期显示对应的彩色底色。
2. **数据准确**：日历正确读取当前登录用户的日记数据；月份切换逻辑（闰年、每月天数、星期对齐）准确无误。
3. **悬停与点击**：鼠标悬停在彩色格子上能看到具体情绪文字；点击彩色格子，页面能自动滚动定位到下方对应的日记卡片。
4. **统计功能**：日历下方能正确计算并显示“本月最常见心情”。
