# 心情日历 UI 优化计划

## 一、需求摘要
用户认为当前的心情日历尺寸过大，要求缩小整体日历尺寸，增加更多圆角元素，并对 UI 细节进行整体优化，使其看起来更加精致、美观，且符合“深夜暗色治愈风”的产品调性。

## 二、当前状态分析
- **DOM 结构**：日历位于 `#tab-history` 内，包含 `.calendar-container` (外层卡片)、`.calendar-header` (头部导航)、`.calendar-weekdays` (星期表头) 和 `.calendar-grid` (日期网格)。
- **现有样式 (`style.css`)**：
  - 外层卡片宽度占满内容区，导致在电脑端显得非常巨大。
  - 日期格子 (`.cal-day`) 采用正方形比例，圆角为 12px，间距为 8px。
  - 情绪背景色使用了降低透明度的纯色填充。

## 三、具体实施方案

### 1. `css/style.css` 核心样式修改
我将通过 CSS 的调整来彻底重塑日历的视觉比例和精致感，无需修改 HTML 结构和 JS 逻辑。

#### A. 尺寸精简与居中 (Smaller & Centered)
- **限制最大宽度**：为 `.calendar-container` 增加 `max-width: 450px;` 和 `margin: 0 auto 40px auto;`，使其在电脑端不再铺满全屏，而是变成一个精致的居中卡片。
- **缩小间距**：将 `.calendar-grid` 的 `gap` 从 `8px` 缩小到 `6px`，让日期排列更紧凑。
- **调整字号**：将 `.cal-day` 的字号缩小至 `14px`，星期表头缩小至 `13px`。

#### B. 极致圆角与形态 (More Rounded)
- **外层大圆角**：将 `.calendar-container` 的圆角提升至 `24px`，与科普卡片保持一致的柔和感。
- **圆形日期格**：将 `.cal-day` 的 `border-radius` 从 `12px` 改为 `50%`，让每一个日期格子都变成完美的**正圆形**。圆形在视觉上比矩形更具治愈感和亲和力。

#### C. 视觉质感优化 (Better Aesthetics)
- **柔和的背景与边框**：
  - 移除带有强烈侵入感的 `.cal-day.has-diary` 的白色内发光边框。
  - 调整情绪底色的透明度，使其更加清透、温润（例如 `rgba(..., 0.7)`）。
- **优化 Hover 动效**：
  - 悬停时的放大比例从 `scale(1.15)` 微微调低到 `scale(1.1)`，避免圆形重叠时显得突兀。
  - 悬停时的阴影改为更加柔和弥散的发光效果 `box-shadow: 0 4px 15px rgba(0, 0, 0, 0.4)`。
- **今日高亮 (Today Highlight)**：
  - 在 JS 中新增一个小逻辑：如果是“今天”，给该格子增加一个特殊的类 `.is-today`，用以显示一个微弱的强调色边框，提升日历的实用性。
- **优化底部总结**：
  - `.calendar-summary` 的顶部虚线边框改为极淡的实线 `1px solid rgba(255,255,255,0.05)`，增加微小的内边距，使其与日历主体融合得更自然。

### 2. `js/app.js` 微调 (添加 Today 标识)
- 在 `renderCalendar` 循环生成日期格子时，获取当前的真实的年、月、日。
- 如果正在生成的格子对应的 `year`、`month` 和 `day` 与“今天”完全一致，则为该 DOM 节点添加 `is-today` 类名。

## 四、假设与决策
- **布局决策**：将日历的最大宽度限制在 450px 并居中，是提升“精致感”最有效的方法。在手机端它依然会自适应屏幕宽度，而在电脑端它将不再显得空旷巨大。
- **形状决策**：将日期格子改为正圆形。在 UI 设计中，密集的网格使用圆形比圆角矩形看起来更加轻盈透气。

## 五、验收标准 (Verification)
1. 刷新“往日回音”页面后，日历整体尺寸明显缩小，在电脑端呈居中显示的精致卡片状。
2. 所有的日期格子变为正圆形，排版紧凑。
3. 带有情绪颜色的格子色彩温润，Hover 时的放大和阴影效果柔和自然。
4. 当天的日期格子（Today）带有一个淡淡的高亮边框标识。
