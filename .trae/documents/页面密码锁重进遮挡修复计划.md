# 页面密码锁重进遮挡修复计划

## Summary

修复“设置页面密码后，重新进入页面时锁屏界面出现布局遮挡、观感不佳并影响输入使用”的问题，重点优化密码锁层的按钮样式作用范围、锁屏容器在不同视口下的排版稳定性，以及解锁前后的界面展示一致性。

## Current State Analysis

- `index.html`
  - 页面密码锁层位于 `#lock-screen`，内部结构为 `.lock-container`、`.lock-password-wrapper`、密码输入框、显示/隐藏密码按钮以及“进入”按钮。
  - 锁屏层与主应用 `#app-container` 并存，通过 `hidden` 类切换显示状态。
- `css/style.css`
  - `#lock-screen` 使用 `position: fixed` 和 `height: 100vh` 全屏居中。
  - `.lock-container button` 当前直接匹配锁屏内所有按钮，会把显示/隐藏密码的小眼睛按钮也一并套用成主按钮样式，和后面的 `.toggle-password-btn` 规则产生冲突，容易形成输入区覆盖或点击区域异常。
  - 锁屏层缺少针对小视口、浏览器地址栏变化、纵向空间不足场景的内边距和滚动兜底，存在内容贴边或被压缩后遮挡的风险。
- `js/app.js`
  - `checkPasswordLock()` 在检测到页面密码后显示 `#lock-screen` 并隐藏 `#app-container`。
  - `unlockApp()` 仅负责解锁显示切换，当前逻辑本身较简单，主要问题更可能来自样式层而不是状态切换流程。

## Proposed Changes

### 1. 收窄锁屏主按钮样式作用范围

- 文件：`css/style.css`
- 修改内容：
  - 将 `.lock-container button` 改为只作用于解锁主按钮，例如改成 `#unlock-btn` 或 `.lock-container > button`。
  - 为锁屏中的 `.toggle-password-btn` 保持独立尺寸、透明背景和自适应宽度，避免覆盖输入框交互区域。
- 目的：
  - 消除“小眼睛按钮覆盖输入区域/挤占布局/视觉异常”的直接根因。

### 2. 优化锁屏层在重进场景下的布局稳定性

- 文件：`css/style.css`
- 修改内容：
  - 为 `#lock-screen` 增加更稳妥的视口高度策略，例如补充 `min-height` / `100dvh` 思路和合适的内边距。
  - 增加 `overflow-y: auto` 之类的兜底，避免较小窗口或移动端浏览器工具栏变化时内容被裁切。
  - 适度优化 `.lock-container` 的宽度、最大宽度、间距、阴影或圆角，使锁屏层视觉层次更统一。
  - 视需要补充锁屏相关的响应式规则，确保输入框、提示文案和按钮纵向排列不拥挤。
- 目的：
  - 解决“重进页面后锁屏区域被遮挡、不美观、影响操作”的整体体验问题，而不是只修一个局部覆盖现象。

### 3. 校验解锁前后展示切换是否需要微调

- 文件：`js/app.js`
- 修改内容：
  - 检查 `checkPasswordLock()` 与 `unlockApp()` 是否需要补充轻量级界面清理，例如确保错误提示状态和输入框内容在显示锁屏时一致。
  - 仅在确有必要时做最小逻辑调整，不改动现有登录/密码锁业务流程。
- 目的：
  - 保证刷新重进、输错密码、再次输入、成功解锁几个关键路径的界面状态稳定。

## Assumptions & Decisions

- 假设用户提到的“密码锁重进之后页面布局遮挡”主要指重新打开页面后显示的锁屏界面，而不是设置页内部表单。
- 决定优先采用 CSS 定向修复，因为当前已能从样式选择器冲突中定位到明确风险点，且这类问题最可能解释“遮挡且影响使用”。
- 决定顺带增强锁屏层的小视口适配，避免修复后只在当前窗口正常、换设备或缩小窗口仍复现。
- 不调整页面密码锁的产品流程，不新增交互步骤，只修复现有体验问题。

## Verification Steps

1. 未设置页面密码时打开页面，确认登录/主界面展示不受影响。
2. 登录后设置页面密码，刷新或重新进入页面，确认锁屏层完整显示，无输入框、图标或按钮互相遮挡。
3. 在锁屏界面点击显示/隐藏密码按钮，确认只切换密码可见性，不覆盖输入区域，也不影响输入焦点。
4. 输入错误密码后检查错误提示位置与间距，确认没有挤压主要输入区域。
5. 输入正确密码后进入主应用，确认主布局恢复正常，没有残留遮罩或错位。
6. 缩小浏览器窗口或在窄屏尺寸下验证锁屏层可完整操作，必要时可滚动，不出现内容被裁切。
